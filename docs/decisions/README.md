# Decisions

Architecture decision records (ADRs) for Versemark (`versemark`).

## Conventions

- Template: [MADR](https://adr.github.io/madr/) (context, decision drivers, considered options with pros/cons, outcome, consequences).
- File names: lowercase, dashes, present-tense imperative verb phrase (per [architecture-decision-record](https://github.com/architecture-decision-record/architecture-decision-record) guidance).
- One decision per file. Supersede rather than rewrite once a decision is implemented.
- Statuses: `proposed`, `accepted`, `superseded by <file>`.
